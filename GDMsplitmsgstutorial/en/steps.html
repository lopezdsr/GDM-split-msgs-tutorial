Overview

This tutorial demonstrates how to create an integration solution that splits a message into multiple messages graphically.

A Mapping node is used to model a graphical data transformation step in the solution. The mapping node has an associated message map, also known as a graphical data map.

The message map is the resource you use to define and implement the data transformation and message split requirements.

Prepare

Click Deploy to deliver the solution from the development environment to the local test environment.

Create the input directory c:\iib\in in your file system. This directory matches the directory that is monitored by the FileInput node.

Create the output directory c:\iib\out in your file system. This directory matches the directory that is monitored by the FileOutput node.

Test the solution

The GDM_Transform_Split1msgintoNmsgs application is shown in the Application Development view of your workspace.

  1. Open the SplitMessageIntoMultipleMessages.msgflow, and click the Flow Exerciser icon to start recording the message path through the flow.
  2. Copy the XML message MultipleCustomerOrdersPerStore.xml into the c:\IIB\in directory. The message is available in the GDM_Transform_Split1msgintoNmsgs project, within the section Other Resources.

After the request message is processed, the message path is automatically highlighted on the message flow.
